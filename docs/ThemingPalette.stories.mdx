# Palette
Each Lightning-UI component can be displayed in one of three visual modes that are referred to as a palette.
All components have three palette options: **neutral**, **brand**, and **inverse**. Each component has a default palette.
For example, Base theme `Buttons` and `ListItems` default to the neutral palette.
A component's palette can be set at the theme or component level.

### Theme level
Each theme may contain a property `componentPalette`. This property can hold one or more properties which correspond to each LUI component's `__componentName`.
You may change the default palette across all LUI components by setting this value in your themes file.
If you want to use a neutral palette on all `ProgressBar` components in the Base or your custom theme you would do this:

```js
componentPalette: {
  ProgressBar: 'neutral'
}
```
The resulting theme object would look like this:

```js
// Inside of themes/base/index.js
export default {
  color,
  layout,
  spacing,
  radius,
  typography,
  animation,
  componentStyle: {},
  componentPalette: {
    ProgressBar: 'neutral'
  }
};
```

In this example, **all** `ProgressBar` components will be using the neutral palette.

### Component level

You may also update this value on a per component basis by setting its palette property. For example:

```js
MyComponent: {
  type: ProgressBar,
  palette: 'inverse'
}
```
In this example, **only** MyComponent's `ProgressBar` will use the inverse palette.
