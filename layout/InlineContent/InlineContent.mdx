# InlineContent

Renders text, icons, and badges in line together

## Usage

Pass in a `content` array containing simple strings or objects with badge or icon strings.

```js
import { InlineContent } from '@lightning/ui';

class Example extends lng.Component {
  static _template() {
    return {
      InlineContent: {
        type: InlineContent,
        content: [
          'example of simple text',
          { icon: 'http://myIconUrl.com/goeshere', title: 'to add voice out content (essentially alt text)', color: '00ff00' },
          { newline: true },
          { badge: 'HD' }
        ]
      }
    };
  }
};
```

## API

### Properties

name|type|required|default|description
--|--|--|--|--
content|string\|array|false|-|list of data to render inline, see [Content Element Properties](#content-element-properties) below
textStyles|object|false|{}|set of nested objects containing [Lightning Text](https://rdkcentral.github.io/Lightning/docs/textures/text) properties which can be referenced in the [Content Element Properties](#content-element-properties) to apply styling to individual text elements
textProperties|object|false|Xfinity theme|font properties to apply to the title
justify|'flex-start'\|'flex-end'\|'center'\|'space-between'\|'space-around'\|'space-evenly'|false|flex-start|alignment of first line flexbox content
iconW|number|false|36|width of all icons
iconH|number|false|36|height of all icons
contentProperties|object|false|{}|object setting flexItem props on all content items
contentSpacing|number|false|8|padding between all content items
contentWrap|boolean|false|false|determines whether the containing flexbox should wrap the content onto the next line
badgeProperties|object|false|Xfinity Badge theme|styling to update the Badge component
combinedLinesHeight|number|0|readonly prop of the total height of all of the contents of the component, accounting for the height of any wrapping that may have occured

### Events

#### $loadedInlineContent():void

Fires when all contents inside the InlineContent component have finished rendering their textures. This is when you will have all of the correct flex properties of the components as well as `finalH` available to be used.

### Content Element Properties
The `InlineContent` component will recognize the following elements inside its `content` param:

string\|object|description
--|--
'Text' \|\| {text: 'Text', style: { styleName: styleValue }} \|\| {text: 'Text', style: 'StyleName' |any stand alone string will be used a pure text, or an object with the properties of `text` and `style` can be passed to override the default text styles, and `style` can be passed a string which references a predefined entry in the [textStyles](#properties) object passed to the component
{ icon: 'pathToIcon', title: 'IconTitle' }|will render an [Icon](/story/elements-icon--text) with the given source and use the title param as the string to announce
{ badge: 'BadgeName' }|will render a [Badge](/story/elements-badge--text)
{ newline: true }|will insert a line break between content if the contentWrap param is true

#### Content String Markup
Additionally, the `content` param will automatically parse a string into the above elements using the following markup:

string markup|parsed element
--|--
`{ICON:title\|url}`|`{ icon: 'url', title: 'title' }`
`{BADGE:title}`|`{ badge: 'title' }`
`{NEWLINE}`|`{ newline: true }`
`{TEXT:text\|styleName}`|`{ text: 'text', style: 'styleName' }`

For example:

```js

import { InlineContent } from '@lightning/ui';

class Example extends lng.Component {
  static _template() {
    return {
      InlineContent: {
        type: InlineContent,
        contentWrap: true,
        textStyles: {
            red: { textColor: getHexColor('FF6194') }
        },
        content: 'This is an {ICON:settings|./assets/icons/settings.png} icon and {BADGE:HD} badge with a{NEWLINE} and {TEXT:red text|red}.'
      }
    };
  }
};
```

will yield

```js
[
  'This is an ',
  { icon: './assets/icons/settings.png', title: 'settings' },
  ' icon and ',
  { badge: 'HD' },
  ' badge with a',
  { newline: true },
  ' and ',
  { text: 'red text', style: 'red' },
  '.'
]
```
