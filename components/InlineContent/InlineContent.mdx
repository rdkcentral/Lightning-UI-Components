# InlineContent

Renders text, icons, and badges in line together

## Usage

Pass in a `content` array containing simple strings or objects with badge or icon strings.

```js
import { InlineContent } from '@lightning/ui';

class Example extends lng.Component {
  static _template() {
    return {
      InlineContent: {
        type: InlineContent,
        content: [
          'example of simple text',
          { icon: 'http://myIconUrl.com/goeshere, title: 'to add voice out content (essentially alt text)', color: '00ff00' },
          { badge: 'HD' }
        ]
      }
    };
  }
};
```

## API

### Properties

name|type|required|default|description
--|--|--|--|--
content|string\|array|false|-|list of data to render inline
textProperties|object|false|Xfinity theme|font properties to apply to the title
justify|'flex-start'\|'flex-end'\|'center'\|'space-between'\|'space-around'\|'space-evenly'|false|flex-start|alignment of first line flexbox content
iconW|number|false|36|width of all icons
iconH|number|false|36|height of all icons
contentProperties|object|false|{}|object setting flexItem props on all content items
contentSpacing|number|false|8|padding between all content items
contentWrap|boolean|false|false|determines whether the containing flexbox should wrap the content onto the next line
badgeProperties|object|false|Xfinity Badge theme|styling to update the Badge component
combinedLinesHeight|number|0|readonly prop of the total height of all of the contents of the component, accounting for the height of any wrapping that may have occured

### Events

#### $loadedInlineContent():void

Fires when all contents inside the InlineContent component have finished rendering their textures. This is when you will have all of the correct flex properties of the components as well as `finalH` available to be used.
