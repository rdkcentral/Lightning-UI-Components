# ListItemToggle

A [`ListItem`](?path=/docs/listitem--basic) with dedicated [`Toggle`](?path=/docs/toggle--basic) functionality

## Usage

`ListItemToggle` can display a `title` and `subtitle`. The toggle itself sits on the right side:

```js
import { ListItemToggle } from '@lightning/ui';

class Example extends lng.Component {
  static _template() {
    return {
      ListItemToggle: {
        type: ListItemToggle,
      }
    };
  }
};
```

<Canvas>
  <Story id="listitemtoggle--basic" />

By default, `ListItemToggle` will toggle on <kbd>Enter</kbd>.

You can override this behavior by setting `onEnter` and use the `toggle()` method to programmatically trigger the toggle and access the checked value. 

```js
const ListItemToggle = this.tag('ListItemToggle');
ListItemToggle.onEnter = listItem => Promise
  .resolve(listItem.toggle())
  .then(checked => /* do something with checked value */)
```

To get the current checked state, use the `isChecked()` method.

```js
if (ListItemToggle.isChecked()) {
  // do something
}
```

## API

### Properties

name|type|readonly|description
--|--|--|--
backgroundType|'fill'\|'float'\|'ghost'|false|color of `ListItem` container.
checked|boolean|false|sets initial `checked` value of Toggle. Use `toggle()` to change value after init
icon|string|false|path to icon image
size|'small'\|'large'\|number|false|width of the `ListItem` container
subtitle|string|false|item metadata info goes here
title|string|false|main item info goes here

### Methods

#### onEnter(thisArg):void

Callback to be executed on <kbd>Enter</kbd> key down event. By default,
calls `this.toggle()`

#### isChecked():boolean

returns the checked value (boolean) for the Toggle

#### toggle():boolean

Toggles the `checked` state and returns the new state.
